<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser input → settings object | Slider value read via .value (always a string) and parsed via parseInt before storage |
| localStorage read → slider initial value | Settings object read at mount time; missing keys fall back to || default |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-05 | Tampering | Slider value read in updateWeights | mitigate | `parseInt(input.value)` converts string to integer; `type="range" min="1" max="50"` enforces browser-level bounds. Combined: stored value is always an integer in [1, 50]. |
| T-10-06 | Tampering | settings object read at mount | accept | `|| default` fallback prevents NaN display if localStorage is cleared or key is absent. Client-side only; no server state affected. |
| T-10-07 | Denial of Service | QuotaExceededError on StorageAdapter.set | accept | Existing `saveAll` already catches QuotaExceededError and logs to console. New sliders add ~60 bytes to the settings object — negligible impact. |
</threat_model>

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip label inline padding (`px-1.5`) |
| sm | 8px | Compact element gaps (`gap-2`, `space-x-2`) |
| md | 16px | Default section padding (`p-4`), element spacing (`space-y-4`) |
| lg | 24px | Section group spacing (`space-y-6`) |
| xl | 32px | — |
| 2xl | 48px | Bottom scroll buffer (`pb-48` safe-area zone) |
| 3xl | 64px | — |

Exceptions:
- Pill chip touch target: minimum 44px height (`py-3` at 12px top/bottom = 44px with 14px font). Source: CONTEXT.md D-09, specifics note "aim for ~44px height."
- Bottom fixed bar: `fixed-safe-bottom` + `p-4` + `env(safe-area-inset-bottom)` — existing pattern, unchanged.
- Back button hit area: `p-2 -ml-2` (32px visible, 40px with padding) — existing pattern, unchanged.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (white) | Page background, court card surfaces |

Accent reserved for:

4. Edit button on most recently played round header — `text-blue-600` text style matching existing "Alternatives" link button
5. Proposed round card header zone background (`bg-blue-50`)

Source: codebase scan of RoundDisplay.js color patterns. CONTEXT.md D-09 specifies blue for Team A, orange for Team B.

## Interaction Contract

### Edit button on proposed round card
- Tap fires `Haptics.light()` then `navigate('/edit/' + round.index)` (0-based index)
- Button is present only when round is NOT played

### Edit button on most recently played round header
- Tap fires `Haptics.light()` then `navigate('/edit/' + round.index)`
- Button is present only for the single most recently played round (`round.index === lastPlayedIdx`)

### Back button in MatchEditor
- Tap fires `navigate('/active')` — never browser history back (D-14)
- No haptic on back (matches existing sub-view pattern — MemberEditor uses plain `<a href="#/">`)

### Nav active state
- `isSession` condition in `router.js` `initRouter` extends to: `hash.startsWith('#/active') || hash.startsWith('#/edit')`
- Session nav item renders `text-blue-600` while in the editor

### No interactivity in Phase 12 (static scaffold only)
- Court chip divs are non-interactive — no click, no drag
- No Confirm / Cancel buttons rendered — those are Phase 13 scope
- Player chips render as visual elements only, not as buttons or drag handles

Source: CONTEXT.md "Not in Phase 12" boundary, D-07, D-14, D-15.

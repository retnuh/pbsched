<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DOM → draft state | data-player-id attributes are read back from the DOM after each drag. These attributes are set by the app itself at render time from session data. Not user-typed input. |
| draft → SessionService | On Confirm, _draft is passed to SessionService.updateRound(). This is internal app data, not external input. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Tampering | chip HTML rendering (teamAChip/teamBChip/benchChip) | mitigate | `escapeHTML()` applied to both player id and player name in every chip helper — prevents XSS if a player name contains HTML special chars |
| T-13-02 | Tampering | reconcileDraftFromDOM reading data-player-id | accept | data-player-id values are set by the app from trusted session data; no user-typed strings reach this attribute; low-value single-user local-first app |
| T-13-03 | Elevation of Privilege | _draft mutated before Confirm | mitigate | All mutations go to _draft (deep copy); original session data untouched until SessionService.updateRound() called on explicit Confirm tap |
| T-13-04 | Denial of Service | SortableJS listeners accumulate on repeated mount/unmount | mitigate | unmount() calls s.destroy() on every SortableJS instance and empties _sortableInstances — prevents duplicate listener accumulation |
</threat_model>

Create `src/services/theme.js` with the following exact implementation (derived from CONTEXT.md locked decisions and PATTERNS.md):

```javascript
/**
 * ThemeService
 * Manages dark/light/auto theme preference — the single point of access for theme operations.
 * Reads and writes preference directly to localStorage at key 'pb:theme'.
 * Callers (Phase 16 Settings toggle) use setMode() and getMode() — never localStorage directly.
 */

const THEME_KEY = 'pb:theme';
const VALID_MODES = ['auto', 'light', 'dark'];

let _mediaQuery = null;
let _mediaListener = null;

export const ThemeService = {
  init() {
    this.applyTheme();
    try {
      _mediaQuery = window.matchMedia('(prefers-color-scheme: dark)');
      _mediaListener = () => {
        if (this.getMode() === 'auto') {
          this.applyTheme();
        }
      };
      _mediaQuery.addEventListener('change', _mediaListener);
    } catch (e) { /* matchMedia unavailable — do nothing */ }
  },

  setMode(mode) {
    if (!VALID_MODES.includes(mode)) return;
    try {
      localStorage.setItem(THEME_KEY, mode);
    } catch (e) { /* quota or private browsing — best effort */ }
    this.applyTheme();
  },

  getMode() {
    try {
      const stored = localStorage.getItem(THEME_KEY);
      return VALID_MODES.includes(stored) ? stored : 'auto';
    } catch (e) {
      return 'auto';
    }
  },

  applyTheme() {
    const mode = this.getMode();
    let isDark;
    if (mode === 'dark') {
      isDark = true;
    } else if (mode === 'light') {
      isDark = false;
    } else {
      // 'auto' or unset — follow system preference
      try {
        isDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
      } catch (e) {
        isDark = false; // safe default: light
      }
    }
    document.documentElement.classList.toggle('dark', isDark);
  },
};
```

Security notes (from RESEARCH.md):
- `getMode()` returns 'auto' for any unrecognized value — defends against a tampered pb:theme key
- `setMode()` validates against VALID_MODES before writing — rejects invalid input silently
- All localStorage and matchMedia calls wrapped in try/catch — safe in private browsing

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage → ThemeService | pb:theme value read from localStorage; could be tampered by other code or manually |
| FOUC script → DOM | Inline script runs with full DOM access before any module loads |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-02-01 | Tampering | ThemeService.getMode() reading pb:theme | mitigate | getMode() returns 'auto' for any value not in VALID_MODES — tampered key falls back safely to system preference |
| T-15-02-02 | Tampering | ThemeService.setMode() writing pb:theme | mitigate | setMode() validates input against VALID_MODES before writing — invalid values are silently rejected |
| T-15-02-03 | Denial of Service | FOUC script localStorage access | mitigate | Entire FOUC script wrapped in try/catch — localStorage unavailable (private browsing) falls back to matchMedia silently; app startup never blocked |
| T-15-02-04 | Denial of Service | ThemeService matchMedia access | mitigate | All matchMedia calls in try/catch — unavailable matchMedia returns isDark=false (safe light default) |
| T-15-02-05 | Information Disclosure | pb:theme localStorage key | accept | Theme preference is non-sensitive; no PII; any code on the same origin can read localStorage anyway |
</threat_model>

  <action>
    ## index.html changes

    1. Find the `<nav>` element. Its current class contains `bg-white/90 backdrop-blur-md border-t border-gray-200`. Add `dark:bg-gray-800/90 dark:border-gray-700` to the class string.

    2. Find all three `<a data-nav="...">` elements. Each currently has `text-gray-400` in its static class. Add `dark:text-gray-500 dark:text-blue-400` to the static class of each `<a>` element.

       Rationale: `dark:text-gray-500` ensures inactive state is visible in dark. `dark:text-blue-400` coexists with the dynamically toggled `text-blue-600` — in dark mode, `dark:text-blue-400` activates when the `.dark` class is on `<html>`, so the active tab shows muted blue instead of vivid blue. No JS changes needed; both classes coexist and the correct one is visible based on dark/light mode.

    ## src/router.js changes

    Read the full router.js. Find the section that toggles nav link active/inactive classes. It looks like:
    ```javascript
    link.classList.toggle('text-blue-600', isActive);
    link.classList.toggle('text-gray-400', !isActive);
    ```

    Add two more toggle calls immediately after, one per line:
    ```javascript
    link.classList.toggle('dark:text-blue-400', isActive);
    link.classList.toggle('dark:text-gray-500', !isActive);
    ```

    This ensures the dark-mode active/inactive classes are dynamically applied in sync with the light-mode classes when the router updates nav state.

    NOTE: The `dark:` prefix on a className string passed to `classList.toggle` is treated as a literal string by the DOM. Tailwind v4 with the `@custom-variant dark (&:where(.dark, .dark *))` definition resolves these as CSS selectors — they work correctly when toggled as class names. This is the same pattern used throughout the codebase.

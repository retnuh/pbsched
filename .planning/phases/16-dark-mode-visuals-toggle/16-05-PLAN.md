<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
    Complete dark mode visual implementation across all 6 views and the nav bar. The Settings page has a new Appearance card (first card) with Auto | Light | Dark segmented control. Zone chips render in blue/orange dark palette. Drag ghost placeholder is visible on dark backgrounds.
  </what-built>
  <how-to-verify>
    Open http://localhost:5173 (or the port shown by `npm run dev`).

    **Step 1: Force dark mode via DevTools**
    - Open browser DevTools → Elements panel → find `<html>` element → manually add class `dark` to it
    - Alternatively: use DevTools → Rendering → "Emulate CSS prefers-color-scheme: dark"

    **Step 2: Check nav bar (bottom of screen)**
    - Nav bar background should be dark (gray-800), not white
    - Inactive nav icons should be muted gray (gray-500), not bright gray
    - Active nav icon should be muted blue (blue-400), not vivid blue-600

    **Step 3: Check Clubs view (ClubManager)**
    - Club cards have dark backgrounds (gray-800)
    - Club names and member counts are readable
    - If no clubs: empty state has dark background with dashed border visible
    - New club form at bottom has dark blue background

    **Step 4: Check Member Editor (MemberEditor)**
    - Member rows have dark backgrounds
    - Start Session card at top has dark blue background (not gray)
    - Roster heading text is readable
    - Name input field has dark background

    **Step 5: Check Session Setup (SessionSetup)**
    - Attendee list rows have dark backgrounds
    - Last played / Never played sub-text is visible (lighter gray on dark)
    - Sticky start bar at bottom has dark background

    **Step 6: Check Rounds view (RoundDisplay)**
    - Round cards have dark backgrounds
    - Team A section shows blue-tinted background (dark blue, not bright blue)
    - Team B section shows orange-tinted background (dark orange, not bright orange)
    - Sitting-out chips are visible on dark background
    - Player names are readable
    - Completed badge text is visible

    **Step 7: Check Match Editor (MatchEditor)**
    - Court cards have dark backgrounds
    - Zone chips: A-side chips are dark blue with light blue text; B-side chips are dark orange with orange text; bench chips are gray
    - Empty slot dashed borders are visible on dark background
    - Bench area has darker gray background
    - Bottom action bar is dark

    **Step 8: Check Settings (Settings)**
    - Appearance card appears FIRST (above Scheduler Optimization)
    - Three buttons: Auto, Light, Dark — the currently active one is highlighted in blue
    - Tap "Dark" button → theme should immediately switch to dark (or stay dark if already dark)
    - Tap "Light" button → theme should immediately switch to light (entire page goes light)
    - Tap "Auto" button → theme should match system preference
    - Tap "Dark" again to return to dark mode
    - All other Settings cards (Scheduler Optimization, Backup & Restore, App Data, About) have dark backgrounds and readable text

    **Step 9: Check Help screen**
    - Help cards have dark backgrounds
    - Section headings are readable in blue
    - Body text is readable (should be light gray, not white-on-white)
    - Back button is visible

    **Step 10: Verify reload persistence**
    - While in dark mode (via Settings toggle set to Dark), reload the page
    - Page should reload in dark mode immediately with no white flash
  </how-to-verify>

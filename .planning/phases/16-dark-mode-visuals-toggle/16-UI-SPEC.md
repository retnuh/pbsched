# Phase 16 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — Tailwind v4 utility classes, plain CSS |
| Preset | not applicable |
| Component library | none (vanilla JS views, innerHTML templates) |
| Icon library | Inline SVG (stroke-based, 24x24 viewBox) |
| Font | System `font-sans` (no custom typeface loaded) |

Source: `index.html`, `src/style.css`, `vite.config.js` (Tailwind v4 via `@tailwindcss/vite`)

Dark mode mechanism: `@custom-variant dark (&:where(.dark, .dark *))` defined in `src/style.css` (delivered by Phase 15). All dark overrides use `dark:` utility prefix or `.dark &` CSS selectors.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline chip gap |
| sm | 8px | Compact element spacing, button inner padding |
| md | 16px | Default card padding (`p-4`), input padding |
| lg | 24px | Section spacing (`space-y-6`) |
| xl | 32px | Layout gaps between major page sections |
| 2xl | 48px | Bottom scroll clearance baseline |
| 3xl | 64px | Safe bottom padding override (`pb-24`, `pb-32`) |

Exceptions:
- Bottom nav height is fixed at `h-20` (80px) — safe-area-aware bottom padding uses `calc(5rem + env(safe-area-inset-bottom))`
- Touch targets on segmented control buttons: minimum 44px height (`py-3` = 12px top + 12px bottom + line-height ≈ 44px total)

Source: `src/style.css` `.fixed-safe-bottom`, `index.html` nav `h-20`, existing Settings.js `py-3` buttons

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (`text-base`) | 400 (regular) | 1.5 |
| Label / small | 14px (`text-sm`) | 400 (regular) | 1.4 |
| Micro / hint | 10px (`text-[10px]`, `text-xs`) | 400 | 1.3 |
| Heading | 24px (`text-2xl`) | 700 (bold) | 1.2 |

Source: inferred from existing views — `text-2xl font-bold` headings, `text-sm` body paragraphs, `text-xs` / `text-[10px]` hint text in Settings.js and RoundDisplay.js.

Dark typography overrides: Tailwind's `dark:` prefix propagates automatically when `.dark` is on `<html>`. The body already declares `dark:text-gray-100` in `index.html`. Per-element overrides are added where the light color is opaque (e.g., `text-gray-500` must become `dark:text-gray-400`).

## Appearance Toggle (Settings — new card)

Source: CONTEXT.md Decisions 3 and 4.

**Component:** 3-button segmented control — `Auto | Light | Dark`

**Placement:** First card in Settings page, above Scheduler Optimization.

**Card structure:**
```
<div class="bg-white dark:bg-gray-800 p-4 rounded-xl shadow-sm border border-gray-100 dark:border-gray-700 space-y-3">
  <h2 class="font-bold text-gray-700 dark:text-gray-200">Appearance</h2>
  <div class="flex rounded-lg overflow-hidden border border-gray-200 dark:border-gray-600">
    <button data-mode="auto">Auto</button>
    <button data-mode="light">Light</button>
    <button data-mode="dark">Dark</button>
  </div>
</div>
```

**Active button state:** `bg-blue-600 text-white font-bold` — matches existing primary button style.

**Inactive button state:** `bg-gray-100 dark:bg-gray-700 text-gray-600 dark:text-gray-300 font-medium` — matches existing muted/secondary style.

**Button dimensions:** Equal-width flex children, `py-2.5` height (minimum 44px touch target), `text-sm font-bold`.

**Behavior:**
- On mount: read `ThemeService.getMode()` → highlight matching button
- On tap: call `ThemeService.setMode(mode)` → update active button highlight → theme applies immediately, no reload
- Import: `import { ThemeService } from '../services/theme.js'`
